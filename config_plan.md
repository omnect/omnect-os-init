Centralize Configuration Handling
Context
Configuration is scattered: /proc/cmdline is parsed twice (once in Config, once in device.rs), OverlayConfig is constructed ad-hoc in main.rs, and feature flags are evaluated inline. Goal: load all config upfront in a single Config struct and pass it through.

Not using OnceLock/singleton — this is a single-threaded PID 1 process where Config is created once and passed to ~5 functions. Explicit passing is simpler and more testable.

#[derive(Debug, Clone)]
pub struct OverlayConfig {
    pub persistent_var_log: bool,
    pub data_mount_options: Option<String>,
}

#[derive(Debug, Clone)]
pub struct Config {
    pub rootfs_dir: PathBuf,
    pub cmdline: CmdlineConfig,
    pub overlay: OverlayConfig,
}
Note: is_release_image stays as cfg!(feature = "release-image") in main() — it's needed before Config::load() succeeds (for error handling if config loading itself fails).
